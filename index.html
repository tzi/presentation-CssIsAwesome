1. Float elements

They are not rendered like an inline or block element.

A good approximation: A floating element is like a block element which has its width that fit to its content.

How to know how your element will be rendered?

  1. display: none
  2. position: absolute/fixed
  3. float: left/right
  4. display: *other values*

2. overflow: hidden

It creates a BFC: Block formating context

The properties of a BFC:
  • Contains his children
  • Does not wrap around floating elements
  • Create a scope for clearing

What elements create a BFC?

  1. overflow: != visible
  2. float: != none
  3. position: absolute/fixed
  4. display: inline-block/table-cell/flex

3. Proportionnal height

allows you to define the height of an element according to the height of his parent 


.element1 {
    height: 200px;
}
.element2 {
    height: 50%;
}




    I'm the parent.
    

        I'm the child.

3. Proportionnal height

It is only working with parent that have a specific height.

The parent height depends of his children height. So, it is to prevent loops in height resolution.
