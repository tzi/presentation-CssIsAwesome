1. Les flottants

Ils ne sont rendus ni comme des éléments inline, ni comme des block.

Une bonne approximation: Un élément flottant est comme un élément block, dont la largeur s'adapte à son contenu.

Comment savoir comment un élément va être rendu ?

  1. display: none
  2. position: absolute / fixed
  3. float: left / right
  4. display: *tout autre valeur*

2. Le débordement

peut créer des BFC: Block formating context

Les propriétés d'une BFC:
  • Il contient ses enfants
  • Il ne s'enroule pas autour des flottants
  • Il crée un nouveau périmètre pour le rétablissement du flux

Comment on crée une BFC ?

  1. overflow: != visible
  2. float: != none
  3. position: absolute/fixed
  4. display: inline-block/table-cell/flex

3. Proportionnal values

allows you to define the height of an element according to the height of his parent 


.green {
    height: 200px;
}
.red {
    height: 50%;
}

3. Proportionnal values

allows you to define a margin according to the size of the element 


.red {
    width: 40%;
    margin-left: 0%;
    margin-top: 0%;
}

3. Proportionnal values

Verticaly box sizing is only working with a parent that have a specific height.

The parent height depends of his children height. So, it is to prevent loops in height resolution.

The margin horizontal & vertical in percent are proportionnal to the parent element width.

5. z-index

is the only way to modify the stack order.

    height: 100px;
}

5. z-index

The render is independant between every stacking context:

  1. z-index < 0
  2. Block elements
  3. Floating elements
  4. Inline elements
  5. z-index > 0

There is different way to create a stacking context:

  1. position: absolute/relative/fixed & z-index != auto
  2. opacity < 1
  3. transform != none
  4. transform-style: preserve-3d
  5. filter != none
  6. clip-path, mask
