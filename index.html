1. Float elements

They are not rendered like an inline or block element.

A good approximation: A floating element is like a block element which has its width that fit to its content.

How to know how your element will be rendered?

  1. display: none
  2. position: absolute/fixed
  3. float: left/right
  4. display: *other values*

2. overflow: hidden

It creates a BFC: Block formating context

The properties of a BFC:
  • Contains his children
  • Does not wrap around floating elements
  • Create a scope for clearing

What elements create a BFC?

  1. overflow: != visible
  2. float: != none
  3. position: absolute/fixed
  4. display: inline-block/table-cell/flex

3. Proportionnal values

Verticaly box sizing is only working with a parent that have a specific height.

The parent height depends of his children height. So, it is to prevent loops in height resolution.

The margin horizontal & vertical in percent are proportionnal to the parent element width.

5. z-index

The render is independant between every stacking context:

  1. z-index < 0
  2. Block elements
  3. Floating elements
  4. Inline elements
  5. z-index > 0

There is different way to create a stacking context:

  1. position: absolute/relative/fixed & z-index != auto
  2. opacity < 1
  3. transform != none
  4. transform-style: preserve-3d
  5. filter != none
  6. clip-path, mask
