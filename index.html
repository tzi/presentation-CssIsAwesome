0. Les marges

Les marges fusionnent avec celles des éléments frères si :

  1. elles sont verticales
  2. il s'agit d'éléments block

Les marges peuvent aussi fusionner entre un élément parent et ses enfants.

1. Les flottants

Ils ne sont rendus ni comme des éléments inline, ni comme des block.

Une bonne approximation: Un élément flottant est comme un élément block, dont la largeur s'adapte à son contenu.

Comment savoir comment un élément va être rendu ?

  1. display: none
  2. position: absolute / fixed
  3. float: left / right
  4. display: *tout autre valeur*

2. Le débordement

peut créer des BFC: Block formating context

Les propriétés d'une BFC:
  • Il contient ses enfants
  • Il ne s'enroule pas autour des flottants
  • Il crée un nouveau périmètre pour le rétablissement du flux

Comment on crée une BFC ?

  1. overflow: != visible
  2. float: != none
  3. position: absolute/fixed
  4. display: inline-block/table-cell/flex

3. Les pourcentages

Le dimensionnement vertical en pourcentage fonctionne seulement si l'élément conteneur a une hauteur précise.

La hauteur du parent dépendant de celle de ces enfants, cela évite les boucles de résolution.

Les marges en pourcentage, quelles soient horizontales ou verticales, sont proportionnelles à la largeur de l'élément conteneur.

5. Le z-index

Le rendu est indépendant entre chaque Stacking Context :

  1. z-index < 0
  2. Block elements
  3. Floating elements
  4. Inline elements
  5. z-index > 0

Il y a plusieurs moyens de créer un Stacking Context :

  1. position: absolute/relative/fixed & z-index != auto
  2. opacity < 1
  3. transform != none
  4. filter != none
  5. clip-path, mask, ...
